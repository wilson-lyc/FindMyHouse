# 功能模块规划

## 1. 房源管理

核心能力：

- 新增、编辑、删除房源。
- 支持手动录入房源基础字段。
- 支持粘贴房源链接并保存来源。
- 支持上传或粘贴图片链接。
- 支持房源状态管理：待查看、已收藏、已联系、已看房、已淘汰、已申请、已签约。
- 支持标签：近地铁、采光好、可养宠、预算压力大、中介费高等。

关键字段：

- 标题、地址、价格、面积、户型、楼层、朝向。
- 押金、付款方式、中介费、可入住日期。
- 联系人、联系方式、来源平台、原始链接。
- 备注、风险点、看房记录。

## 2. 地点管理

用户可以维护多个关键地点：

- 公司
- 学校
- 伴侣或室友公司
- 常去地点
- 家庭成员住址

每个地点可设置权重。例如，公司通勤权重 60%，伴侣公司通勤权重 30%，健身房或商圈权重 10%。

## 3. 双模式地图工作台

地图是产品的核心视图，但阶段 2 后的工作台不只保留一种布局。应用支持“列表模式”和“大屏地图模式”：前者适合高密度管理和字段比较，后者适合空间关系判断。用户可以随时切换，且筛选条件、选中房源和地图视野筛选状态保持一致。

- 展示房源点位。
- 展示关键地点点位。
- 支持列表模式：列表为主，地图为右侧小窗。
- 支持大屏地图模式：地图为主画布，列表、筛选、详情、关键地点和摘要以浮动面板呈现。
- 支持两种模式随时切换。
- 点击房源点位打开右侧详情面板。
- 点击列表项高亮地图点位。
- 根据价格、状态、推荐分数使用不同颜色或图标。
- 支持地图框选或当前视野筛选。
- 支持大屏地图模式下的左侧房源列表面板、右侧详情面板和底部分析面板。
- 支持面板折叠，让用户随时回到以地图为主的空间判断。
- 支持显示通勤路线。
- 支持热力层或生活便利性层的后续扩展。

更多交互设计见 [双模式地图工作台设计](./map-workbench-design.md)。

## 4. 通勤分析

通勤分析需要支持多种方式：

- 步行
- 骑行
- 公交
- 地铁
- 驾车
- 混合通勤

分析指标：

- 距离
- 预计通勤时间
- 换乘次数
- 步行距离
- 预计费用
- 高峰期和非高峰期差异
- 雨天或夜间体验的主观备注

## 5. 筛选与排序

基础筛选：

- 租金区间
- 面积区间
- 户型
- 状态
- 标签
- 可入住时间
- 到某地点通勤时间上限

推荐排序：

- 综合推荐分
- 租金从低到高
- 通勤时间从短到长
- 面积从大到小
- 最近更新
- 用户自定义权重排序

## 6. 房源对比

对比视图用于最终决策：

- 支持选择 2 到 5 套房源横向对比。
- 展示价格、面积、每平米租金、通勤时间、押金、中介费、标签、风险点。
- 高亮最优项和明显劣势项。
- 支持 Agent 生成对比总结。

## 7. 看房记录

看房后可以记录：

- 实际照片
- 实际感受
- 噪音、采光、楼道、电梯、周边环境
- 房东或中介沟通情况
- 合同注意事项
- 是否继续考虑

这些记录应参与 Agent 的后续推荐。

## 8. Agent 推荐与分析

Agent 模块提供：

- 房源摘要：把杂乱备注整理成结构化优缺点。
- 风险识别：识别价格异常、押金异常、交通描述不清等问题。
- 个性化推荐：结合预算、通勤、面积、生活偏好排序。
- 对比报告：解释为什么 A 比 B 更适合。
- 看房问题清单：针对某套房生成看房时应确认的问题。
- 决策复盘：当用户淘汰房源时总结原因，更新偏好模型。

## 9. 数据导入与导出

MVP 先支持：

- CSV 导入
- CSV 导出
- JSON 备份

后续扩展：

- 浏览器剪藏插件
- 房源网页解析
- 截图 OCR
- 表格同步

## 10. 设置与偏好

用户可配置：

- 预算上限和理想区间。
- 通勤时间上限。
- 各地点权重。
- 偏好标签。
- 排除条件。
- 地图服务 API Key。
- Agent Provider 和模型。
