# 项目 Roadmap

## 阶段 0：项目启动与基础设计

目标：完成项目方向、技术方案和文档基础。

交付物：

- Git 仓库初始化。
- `dev` 分支创建。
- 根目录 `README.md` 项目入口。
- 根目录 `.gitignore` 仓库基础配置。
- 项目文档目录建立。
- 总体设计、架构、功能模块、数据模型、Agent 方案和 Roadmap。
- 阶段 0 验收清单。

验收标准：

- 文档可以指导后续工程初始化。
- 技术栈、模块边界和 MVP 范围清晰。
- 阶段 1 的前后端、数据库和房源 CRUD 任务可以从文档中直接拆分。

## 阶段 1：工程骨架与基础数据

目标：搭建可运行的前后端工程。

主要任务：

- 初始化 Vue 3 + TypeScript + Vite 前端。
- 初始化 Node.js + TypeScript 后端。
- 选择 Fastify 或 Express。
- 接入 SQLite 和迁移工具。
- 建立基础目录结构和开发脚本。
- 实现房源 CRUD API。
- 实现前端房源列表与表单。

验收标准：

- 本地一条命令可启动前后端开发环境。
- 用户可以创建、查看、编辑和删除房源。
- SQLite 中能持久化房源数据。

## 阶段 2：双模式地图工作台

目标：提供“列表模式”和“大屏地图模式”两种可随时切换的租房决策工作台。列表模式保持高密度管理效率，大屏地图模式强化空间关系判断；两种模式共享房源列表、筛选、关键地点、选中态和地图视野筛选状态。

当前进展：

- 关键地点 CRUD API 已实现。
- 高德 Web 服务地理编码接口已接入。
- 前端高德 JS SDK 加载已接入。
- 房源和关键地点已具备点位展示基础。
- 地图视野范围筛选已具备基础能力。
- 列表模式和大屏地图模式已可切换。
- 列表模式已取消固定窄版容器，页面可充分使用屏幕宽度。
- 大屏地图模式已建立浮动筛选条、左侧房源列表、右侧详情、关键地点面板和底部统计摘要。
- 房源详情/编辑弹窗已支持内嵌地图点选坐标，用户无需手动输入经纬度。
- 房源详情/编辑弹窗已按基础信息、位置定位、租金付款、额外费用、房屋条件、来源备注分区。
- 房源详情/编辑弹窗顶部已提供等分的分区跳转导航，仅用于滚动跳转，不隐藏表单字段。
- 内嵌高德地图选择器已做局部层级隔离，避免地图控件和水印穿透覆盖弹窗导航。

设计调整：

- 主页面保留列表为主、小地图为辅的当前模式。
- 新增大屏地图模式，房源列表、筛选器、关键地点管理、详情摘要和分析统计作为浮动面板出现。
- 用户可以在两种模式间自由切换，切换时保留筛选条件和选中房源。
- 地图点位、列表项和详情面板需要双向联动。
- 房源录入和编辑按决策维度分区，长表单优先保持连续上下文，并通过顶部导航提高跳转效率。
- 后续通勤路线、对比和 Agent 分析都应复用双模式工作台的面板体系。

主要任务：

- 完成双模式地图工作台信息架构设计。已完成
- 保留当前列表工作台布局，新增大屏地图布局。已完成
- 建立模式切换、顶部工具条、左侧房源列表面板、右侧详情面板和底部分析面板。已完成基础版本
- 保留并整合地点管理能力。已完成基础版本
- 完善高德地图 SDK 接入和错误提示。已完成基础版本
- 实现房源地址和关键地点地址地理编码。已完成基础版本
- 支持房源通过内嵌地图点选校准坐标。已完成基础版本
- 优化房源详情/编辑弹窗的长表单信息架构、分区导航和滚动体验。已完成基础版本
- 在地图上展示房源和关键地点。已完成基础版本
- 支持地图点位点击打开详情面板。已完成基础版本
- 支持列表项点击高亮地图点位。已完成基础版本
- 实现地图视野范围内筛选，并提供明确的开启、取消和状态提示。已完成基础版本，语义待打磨
- 为阶段 3 的通勤路线和阶段 5 的 Agent 分析预留面板位置。

阶段 2 剩余收尾：

- 打磨地图点位、列表项、详情面板之间的选中态和滚动定位。
- 明确地图视野筛选的开关机制，避免地图移动后自动改变列表造成困惑。
- 强化无坐标房源的提示、筛选和批量定位入口。
- 优化大屏地图模式在移动端的抽屉交互。
- 补充关键地点的地图点选坐标能力，与房源详情/编辑弹窗体验保持一致。
- 检查所有高德地图实例在弹窗、面板和浮层中的层级边界，避免第三方地图控件覆盖自有 UI。
- 为房源详情分区补充必要的只读摘要入口，例如总月成本、首期成本和缺失字段提醒。

验收标准：

- 用户可以维护公司等关键地点。已达成基础版本
- 用户进入应用后可以使用列表模式，并可随时切换到大屏地图模式。已达成基础版本
- 房源详情/编辑弹窗可按分区录入和跳转，地图点选不会遮挡弹窗导航。已达成基础版本
- 房源和关键地点可稳定显示在地图上。已达成基础版本
- 地图点位、房源列表和详情面板可以联动。已达成基础版本，体验待打磨
- 地图视野筛选与列表筛选可以联动。已达成基础版本，开关语义待打磨
- 无坐标房源有明确提示和定位入口。待强化
- 页面布局能支持后续通勤、对比和 Agent 分析能力扩展。

阶段 2 收口建议：

1. 先完成关键地点地图点选和无坐标房源提示，补齐定位体验闭环。
2. 再打磨地图视野筛选开关、选中态联动和移动端抽屉。
3. 最后做阶段 3 前置清理：路线图层占位、详情通勤摘要占位、通勤缓存字段检查。

## 阶段 3：通勤分析

目标：建立租房决策最关键的通勤指标。

主要任务：

阶段 3.1：路线服务与缓存

- 封装高德路线规划服务。
- 支持至少一种通勤方式，优先公交或驾车，具体取决于目标城市地图服务能力。
- 以房源和关键地点为输入计算路线。
- 保存通勤计算结果，避免重复调用外部 API。
- 记录路线计算失败原因和更新时间。

阶段 3.2：通勤摘要展示

- 在房源详情、列表和地图信息窗中展示到默认关键地点的通勤时间。
- 在底部分析面板展示当前候选集的最短、最长和平均通勤时间。
- 在房源详情分区中展示通勤缺失状态，引导用户补齐坐标或关键地点。

阶段 3.3：筛选排序与路线图层

- 支持按通勤时间筛选和排序。
- 支持在地图上展示选中房源到关键地点的路线。
- 支持用户手动刷新某套房源的通勤结果。

验收标准：

- 用户可以看到每套房到关键地点的通勤时间。
- 通勤结果可缓存，避免重复调用外部 API。
- 房源列表支持通勤排序。
- 无坐标房源、无关键地点、路线计算失败都有明确提示。

## 阶段 4：筛选、标签与对比

目标：让用户可以高效缩小候选范围。

主要任务：

- 实现租金、面积、户型、状态、额外费用和可入住日期筛选。
- 实现标签管理和房源标签。
- 实现收藏和淘汰流程。
- 在房源详情中沉淀总月成本、首期成本等预算辅助指标。
- 实现 2 到 5 套房源横向对比。
- 对比视图高亮明显优势和劣势。

验收标准：

- 用户可以从几十套房源中快速筛选候选。
- 用户可以生成清晰的候选房源对比表。

## 阶段 5：Agent MVP

目标：引入 Agent，完成基础智能分析能力。

主要任务：

- 设计 Agent Service。
- 接入 LLM Provider。
- 实现单房源摘要。
- 实现风险提示。
- 实现多房源推荐排序解释。
- 实现看房问题清单。
- 保存 Agent 分析历史。

验收标准：

- Agent 输出基于真实房源数据。
- 推荐结论包含明确依据。
- 缺失信息会被标注，而不是被编造。

## 阶段 6：看房记录与反馈闭环

目标：让用户的实际看房体验参与推荐。

主要任务：

- 实现看房记录模块。
- 支持照片、评分、优缺点、待确认问题。
- Agent 能结合看房记录更新分析。
- 淘汰房源时记录淘汰原因。
- 基于用户反馈调整推荐权重。

验收标准：

- 看房前、看房后、最终决策之间形成数据闭环。
- Agent 推荐能体现用户反馈变化。

## 阶段 7：数据导入导出

目标：降低数据录入成本，提升数据可迁移性。

主要任务：

- CSV 导入房源。
- CSV 导出房源。
- JSON 备份与恢复。
- 支持粘贴多行文本后由 Agent 辅助结构化。

验收标准：

- 用户能从表格迁移已有房源数据。
- 用户能备份自己的数据。

## 阶段 8：增强能力

目标：从 MVP 走向更完整的租房决策系统。

候选方向：

- 浏览器剪藏插件。
- 房源网页自动解析。
- 截图 OCR。
- POI 生活便利性分析。
- 多人协作找房。
- 房源价格异常检测。
- 合同条款检查清单。
- 移动端适配或 PWA。
- 从 SQLite 迁移到 PostgreSQL 的部署版本。

## 优先级建议

当前优先级建议：

1. 阶段 2 收口：关键地点点选、无坐标提示、视野筛选开关、移动端抽屉。
2. 阶段 3 通勤 MVP：路线服务、缓存、列表和详情通勤摘要。
3. 阶段 4 基础筛选：租金、面积、户型、额外费用、可入住日期和标签。
4. 阶段 4 对比 MVP：2 到 5 套房源横向对比和预算/通勤优势高亮。
5. 阶段 5 Agent MVP：单房源摘要、风险提示、看房问题清单和多房源排序解释。

MVP 必须覆盖：

1. 房源 CRUD。
2. 地点管理。
3. 地图展示。
4. 通勤计算。
5. 基础筛选与排序。
6. Agent 单房源摘要和多房源推荐解释。

可以延后：

- 自动网页解析。
- 多人协作。
- OCR。
- 高级 Agent 记忆。
- 移动 App。
