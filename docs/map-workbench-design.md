# 双模式地图工作台设计

## 设计目标

阶段 2 的核心界面调整为“双模式工作台”：用户既可以使用“列表为主、地图为辅”的高密度管理模式，也可以随时切换到“大屏地图”的空间判断模式。两种模式共享同一套房源、筛选、关键地点、选中态和地图视野筛选状态。

这个方向符合 FindMyHouse 的产品原则：租房决策既需要表格和列表带来的管理效率，也需要地图带来的位置关系判断。用户不应被迫在两者之间二选一，而是可以根据当前任务自由切换。

## 信息架构

主页面采用两种可切换模式：

- 列表模式：保留当前工作台结构，房源列表、筛选器和统计摘要是主视图，地图和关键地点管理位于右侧小窗。
- 大屏地图模式：地图成为主画布，房源列表、筛选器、关键地点、详情和统计以浮动面板叠加在地图上。

两种模式共用：

- 顶部工具条：承载模式切换、新增房源和后续全局操作。
- 房源筛选器：搜索、状态筛选，以及后续预算、面积、标签和通勤筛选。
- 地图画布：展示房源点位、关键地点点位、选中态、视野范围和后续通勤路线。
- 房源列表：展示当前筛选结果，并与地图点位联动。
- 详情面板：展示当前选中房源、关键地点或路线的详情。
- 分析摘要：展示候选数量、平均租金、收藏数量和视野筛选状态。

## 模式布局

### 列表模式

用途：

- 快速录入、维护、筛选和批量浏览房源。
- 适合用户需要比较大量字段、频繁编辑和清理数据的时候。

桌面端布局：

- 顶部显示统计摘要、筛选器和模式切换。
- 左侧主区域显示房源表格。
- 右侧显示小地图和关键地点管理面板。
- 地图点位点击可选中房源，列表行点击也会同步打开地图信息窗。

### 大屏地图模式

用途：

- 以位置关系为主进行判断。
- 适合用户查看房源与公司、学校、交通枢纽和生活圈的关系。

桌面端布局：

- 地图占据主要视口。
- 顶部浮动筛选条显示搜索、状态筛选和刷新动作。
- 左侧浮动房源列表展示当前筛选结果。
- 右侧浮动详情面板展示当前选中房源。
- 右下浮动关键地点面板用于维护公司等地点。
- 底部浮动摘要显示候选数量、平均租金、收藏数量和地图视野筛选状态。

移动端布局：

- 列表模式优先保证表格和筛选可用。
- 大屏地图模式中，房源列表变成底部抽屉。
- 详情和关键地点管理后续可作为抽屉打开。
- 地图控件收敛为少量按钮。

## 核心面板

### 房源列表面板

用途：

- 快速浏览当前筛选结果。
- 支持按价格、更新时间、通勤时间、推荐分排序。
- 点击列表项时同步高亮地图点位并打开详情。

内容：

- 搜索框。
- 状态、预算、面积、户型等基础筛选。
- 当前是否启用地图视野筛选。
- 房源卡片列表：标题、地址、租金、面积、状态、距离或通勤摘要。

交互：

- 大屏地图模式中面板可折叠，折叠后只保留一个列表按钮和结果数量。
- 滚动列表不会移动地图。
- 地图移动后，如果视野筛选开启，列表自动刷新。

### 详情面板

用途：

- 展示房源或关键地点的上下文信息。
- 承载编辑、删除、收藏、淘汰、加入对比等操作。

房源详情内容：

- 标题、地址、租金、面积、户型、状态。
- 坐标和定位质量。
- 到关键地点的通勤摘要，阶段 3 完成后补齐。
- 备注、来源链接、风险提示和看房记录入口。

关键地点详情内容：

- 名称、类别、地址、坐标。
- 权重和未来通勤计算设置。

### 底部分析面板

用途：

- 给用户提供地图当前状态的总结。
- 在后续阶段承载通勤、对比和 Agent 分析结果。

阶段 2 内容：

- 当前视野内房源数量。
- 有坐标和无坐标的房源数量。
- 平均租金、最低租金、最高租金。
- 关键地点数量。

阶段 3 后扩展：

- 到公司最短通勤、最长通勤和平均通勤。
- 当前选中房源的多方式通勤对比。

阶段 5 后扩展：

- Agent 对当前候选集的推荐解释。
- 明显风险和缺失信息提醒。

## 地图点位规则

房源点位：

- 用价格或状态做主要视觉编码。
- 收藏、淘汰、已联系等状态使用不同颜色或图标。
- 点位 hover 显示简短标题和租金。
- 点位 click 打开右侧详情面板。

关键地点点位：

- 使用不同于房源的形状或颜色。
- 公司、学校、交通枢纽、常用地点可使用不同图标。
- 关键地点始终优先可见，不被房源点位淹没。

无坐标房源：

- 不显示在地图上。
- 在列表中提示“待定位”。
- 提供批量地理编码或逐条定位入口。

## 关键工作流

### 切换模式

1. 用户在顶部工具条选择“列表模式”或“大屏地图”。
2. 系统保留当前筛选条件、选中房源和地图视野筛选状态。
3. 用户可继续在任一模式中选择房源、编辑房源或维护关键地点。

### 新增房源

1. 用户点击新增房源。
2. 在表单中输入地址和基础信息。
3. 系统调用高德地理编码写入坐标。
4. 保存后地图立即出现新点位，并可选中新房源。

### 地图视野筛选

1. 用户移动或缩放地图后，系统可将当前地图视野作为列表筛选条件。
2. 列表模式和大屏地图模式共用这组视野筛选条件。
3. 顶部、列表上方或底部摘要显示筛选状态。
4. 用户可以一键取消视野筛选，列表回到全量筛选结果。

### 查看房源

1. 用户点击地图点位或列表项。
2. 地图高亮该点位。
3. 大屏地图模式中右侧详情面板展开，列表模式中小地图打开信息窗。
4. 用户可继续编辑、加入对比或查看未来通勤分析。

### 维护关键地点

1. 用户从右侧面板或大屏地图浮动面板进入关键地点管理。
2. 新增公司等地点并定位。
3. 地点点位固定显示在地图上。
4. 阶段 3 中关键地点将作为路线规划终点。

## 阶段 2 实现拆分

阶段 2.1：双模式工作台壳层

- 保留列表模式，新增大屏地图模式。
- 建立模式切换、浮动筛选条、左侧房源面板、右侧详情面板、关键地点面板和底部摘要的基础结构。
- 保留现有房源 CRUD 和关键地点 CRUD 能力。

阶段 2.2：点位与面板联动

- 地图点位点击打开详情面板。
- 列表点击高亮地图点位。
- 点位、列表和选中态保持同步。

阶段 2.3：视野筛选与定位体验

- 完善地图视野筛选开关。
- 强化无坐标房源提示。
- 支持房源和关键地点的定位失败反馈。

阶段 2.4：信息密度和移动端适配

- 优化浮动面板、抽屉和响应式布局。
- 控制地图上点位密度，避免遮挡。
- 为后续通勤路线和 Agent 分析预留面板位置。

## 暂不在阶段 2 处理

- 通勤路线计算和路线图层。
- POI 生活便利性分析。
- 多房源横向对比完整视图。
- Agent 推荐结论。
- 地图聚合、热力图和复杂图层管理。
