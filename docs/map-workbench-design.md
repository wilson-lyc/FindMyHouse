# 全屏地图工作台设计

## 设计目标

阶段 2 的核心界面从“列表为主、地图为辅”调整为“地图为主画布、面板承载操作”。用户进入应用后第一眼看到的是全屏地图，房源、关键地点、通勤关系和筛选结果都围绕地图展开。

这个方向符合 FindMyHouse 的产品原则：租房决策首先是位置关系决策，列表、表单和分析都应该服务于地图上的空间判断。

## 信息架构

主页面采用全屏地图工作台：

- 地图画布：占满整个浏览器视口，展示房源点位、关键地点点位、选中态、视野范围和后续通勤路线。
- 顶部工具条：承载全局搜索、城市/区域、保存、导入、设置和状态提示。
- 左侧房源面板：展示房源列表、基础筛选、排序和批量选择。
- 右侧详情面板：展示当前选中房源、关键地点或路线的详情。
- 底部分析面板：展示通勤摘要、对比结果、Agent 结论或当前筛选统计，可折叠。
- 地图内快捷控件：定位、缩放、图层、视野筛选开关、显示/隐藏面板。

## 默认布局

桌面端默认状态：

- 地图全屏铺底。
- 左侧面板宽度约 360px，显示房源列表和筛选器。
- 右侧面板默认收起，用户点击房源点位或列表项后展开。
- 底部面板默认半收起，仅显示候选数量、平均租金、已收藏数量和地图视野筛选状态。
- 新增房源、关键地点和设置入口放在顶部工具条或左侧面板顶部。

移动端默认状态：

- 地图仍然是第一屏。
- 房源列表变成底部抽屉，可在 25%、55%、90% 高度之间切换。
- 详情面板作为全屏抽屉打开。
- 地图控件收敛为少量图标按钮。

## 核心面板

### 房源列表面板

用途：

- 快速浏览当前筛选结果。
- 支持按价格、更新时间、通勤时间、推荐分排序。
- 点击列表项时同步高亮地图点位并打开详情。

内容：

- 搜索框。
- 状态、预算、面积、户型等基础筛选。
- 当前是否启用地图视野筛选。
- 房源卡片列表：标题、地址、租金、面积、状态、距离或通勤摘要。

交互：

- 面板可折叠，折叠后只保留一个列表按钮和结果数量。
- 滚动列表不会移动地图。
- 地图移动后，如果视野筛选开启，列表自动刷新。

### 详情面板

用途：

- 展示房源或关键地点的上下文信息。
- 承载编辑、删除、收藏、淘汰、加入对比等操作。

房源详情内容：

- 标题、地址、租金、面积、户型、状态。
- 坐标和定位质量。
- 到关键地点的通勤摘要，阶段 3 完成后补齐。
- 备注、来源链接、风险提示和看房记录入口。

关键地点详情内容：

- 名称、类别、地址、坐标。
- 权重和未来通勤计算设置。

### 底部分析面板

用途：

- 给用户提供地图当前状态的总结。
- 在后续阶段承载通勤、对比和 Agent 分析结果。

阶段 2 内容：

- 当前视野内房源数量。
- 有坐标和无坐标的房源数量。
- 平均租金、最低租金、最高租金。
- 关键地点数量。

阶段 3 后扩展：

- 到公司最短通勤、最长通勤和平均通勤。
- 当前选中房源的多方式通勤对比。

阶段 5 后扩展：

- Agent 对当前候选集的推荐解释。
- 明显风险和缺失信息提醒。

## 地图点位规则

房源点位：

- 用价格或状态做主要视觉编码。
- 收藏、淘汰、已联系等状态使用不同颜色或图标。
- 点位 hover 显示简短标题和租金。
- 点位 click 打开右侧详情面板。

关键地点点位：

- 使用不同于房源的形状或颜色。
- 公司、学校、交通枢纽、常用地点可使用不同图标。
- 关键地点始终优先可见，不被房源点位淹没。

无坐标房源：

- 不显示在地图上。
- 在左侧面板中单独提示“待定位”。
- 提供批量地理编码或逐条定位入口。

## 关键工作流

### 新增房源

1. 用户点击新增房源。
2. 在浮动表单或右侧面板中输入地址和基础信息。
3. 系统调用高德地理编码写入坐标。
4. 保存后地图立即出现新点位，并选中新房源。

### 地图视野筛选

1. 用户打开“按当前视野筛选”。
2. 移动或缩放地图后，列表只显示视野内房源。
3. 顶部或底部显示筛选状态。
4. 用户可以一键取消视野筛选，列表回到全量筛选结果。

### 查看房源

1. 用户点击地图点位或列表项。
2. 地图高亮该点位。
3. 右侧详情面板展开。
4. 用户可继续编辑、加入对比或查看未来通勤分析。

### 维护关键地点

1. 用户从顶部工具条或地图控件进入关键地点管理。
2. 新增公司等地点并定位。
3. 地点点位固定显示在地图上。
4. 阶段 3 中关键地点将作为路线规划终点。

## 阶段 2 实现拆分

阶段 2.1：地图工作台壳层

- 将主页面改为全屏地图布局。
- 建立顶部工具条、左侧面板、右侧面板、底部面板的基础结构。
- 保留现有房源 CRUD 和关键地点 CRUD 能力。

阶段 2.2：点位与面板联动

- 地图点位点击打开详情面板。
- 列表点击高亮地图点位。
- 点位、列表和选中态保持同步。

阶段 2.3：视野筛选与定位体验

- 完善地图视野筛选开关。
- 强化无坐标房源提示。
- 支持房源和关键地点的定位失败反馈。

阶段 2.4：信息密度和移动端适配

- 优化面板折叠、抽屉和响应式布局。
- 控制地图上点位密度，避免遮挡。
- 为后续通勤路线和 Agent 分析预留面板位置。

## 暂不在阶段 2 处理

- 通勤路线计算和路线图层。
- POI 生活便利性分析。
- 多房源横向对比完整视图。
- Agent 推荐结论。
- 地图聚合、热力图和复杂图层管理。
